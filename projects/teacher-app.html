À propos du projet & de mon rôle

En tant que Designer Produit bénévole, j’ai rejoint ce projet initié par l’Institut Malik Ibn Anas. L’objectif était de faciliter la communication entre parents et enseignants dans une école coranique. Le besoin est né d’une douleur terrain concrète : les professeurs perdaient énormément de temps à contacter les parents individuellement par WhatsApp, ce qui nuisait à leur concentration et à leur gestion de classe.

  • ✅ Contexte réel : enseignants bénévoles, parents très impliqués, école sans budget tech.
  • ✅ Conception de A à Z : ateliers, wireframes, prototypes, tests.
  • ✅ Projet en production : l’app est utilisée depuis mars 2025.

Problème

L’école utilisait WhatsApp et des tableaux Excel pour transmettre les informations : retards, absences, devoirs… mais cela générait des oublis, des confusions et un surmenage côté enseignants. Les parents étaient aussi frustrés par le manque de visibilité. Il fallait centraliser la communication dans une interface simple, claire, et adaptée à tous les profils.

Enjeux & Contraintes

Le contexte bénévole et associatif impliquait :

  • ✅ Aucun budget, donc outils gratuits (Figma, Notion, Google Sheets...)
  • ✅ Professeurs non technophiles, donc interface ultra-intuitive
  • ✅ Pas d’application mobile native, donc version web responsive

Recherche utilisateur

Nous avons réalisé :

  • ✅ Des entretiens avec les enseignants et les responsables pédagogiques
  • ✅ Un benchmark de solutions existantes (apps de classe, gestion scolaire…)
  • ✅ Une priorisation des besoins selon la méthode MoSCoW

About the project & my role

As a volunteer Product Designer, I joined this project initiated by the Malik Ibn Anas Institute. The goal was to facilitate communication between parents and teachers in a Quranic school. The need came from a very concrete pain point: teachers were wasting a lot of time contacting parents one by one via WhatsApp, which disrupted their focus and class management.

  • ✅ Real context: volunteer teachers, highly involved parents, no tech budget.
  • ✅ End-to-end design: workshops, wireframes, prototypes, user testing.
  • ✅ Live project: the app has been used since March 2025.

Problem

The school was using WhatsApp and Excel sheets to share information: lateness, absences, homework... but this led to oversights, confusion, and overload for teachers. Parents were also frustrated with the lack of visibility. The need was to centralize communication in a simple, clear interface that works for everyone.

Challenges & Constraints

The nonprofit context brought the following constraints:

  • ✅ Zero budget → free tools (Figma, Notion, Google Sheets...)
  • ✅ Non-tech-savvy teachers → ultra-intuitive interface
  • ✅ No native mobile app → responsive web version

User Research

We conducted:

  • ✅ Interviews with teachers and educational staff
  • ✅ A benchmark of existing tools (classroom apps, school management platforms…)
  • ✅ Prioritization using the MoSCoW method

Results & Feedback

Methodology

  • ✅ Real scenarios: tested with everyday use cases.
  • ✅ Iterative feedback: improvements based on feedback from both teachers and parents.

User Feedback

Both sides praised the intuitive interface and time-saving aspect. Parents loved the centralization, and teachers appreciated how easy it was to update information.

Challenges Faced

  • ✅ Onboarding: ensuring accessibility for non-tech-savvy users.
  • ✅ Compatibility: adapting the design for older devices.

Impact

  • ✅ Fewer misunderstandings between teachers and families.
  • ✅ Time saved for teaching staff.
  • ✅ No more scattered or paper-based tools.

Tools Used

  • ✅ MakeMyPersona (Hubspot) for personas.
  • ✅ Photoshop for screens and branding.
  • ✅ Figma for wireframes and UI design.
  • ✅ Trello for agile collaboration.
  • ✅ Microsoft (Forms, Teams, Whiteboard) for testing and brainstorming.
